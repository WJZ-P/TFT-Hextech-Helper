# Changelog

All notable changes to this project will be documented in this file.

## [1.6.0] - 2026-04-21
### S17 星神赛季适配
- 新增 S17 赛季完整数据：棋子池、羁绊（种族 / 职业）、装备（基础散件、成装、神器、辅助装、专属）以及对应的棋子图标与模板。
- 首页模式选择：下线 S4「瑞兽闹新春」；将原「英雄联盟传奇」入口改为「S17 星神」，作为当前主赛季。
- 新增 15 个 S17 主流阵容，支持在阵容页直接勾选使用。
- 新增 3 种 S17 专属回合识别与自动处理：
  - 星神选择（2-4 / 3-4 / 4-4）：从 2 个候选槽位中自动选择。
  - 大恩赐（4-7）：自动点击右下角按钮领取。
  - 小型恩赐（5-4 / 6-4 / 7-4）：从 4 个装备槽位中自动选择。
- 新增「时空核心（未来战士核心）」特殊棋子处理：在备战席识别到时空核心后，直接卖出换取经验值，流程与锻造器并列共用一套「右键弹浮窗」识别机制。

### 架构升级
- 新增统一赛季注册中心 `SeasonRegistry`，集中管理 `CURRENT_SEASON`、棋子 / 装备 / 羁绊数据、模板目录等映射。换赛季只需改一处常量，业务代码不再散落硬编码。
- 新增赛季白名单过滤：初始化阵容时自动剔除非当前赛季的选中阵容，避免历史赛季阵容（如 S16）混入匹配算法导致误锁定。
- 新增"不可售卖棋子"保护名单：魔像、训练假人、迷你黑洞、未来战士核心等特殊单位不会被自动卖出流程误伤。

### UI 优化
- 首页模式卡片 hover 浮窗改为"估计占位 + rAF 真实尺寸修正"两阶段定位，消除切换模式时的浮窗位置偏移。
- 阵容页头像统一切换到稳定的国内 CDN 源，不再区分赛季来源，加载速度与稳定性提升。

### 修复
- 修复前端 `LineupsPage` 在部分情况下因 `TFT_16_CHESS` 未导出而报错的问题。
- 修复 `GameStateManager.addEquipToUnit` 装备类型不完整（缺失 `englishName` / `equipId` / `formula`）导致的类型错误，改为从赛季装备字典查询完整 `TFTEquip` 记录再写入。
- 修复 OP.GG 阵容转换脚本对召唤物（如小木灵、迷你黑洞、未来战士核心）的误判，采用"字典白名单优先 + 正则过滤兜底"策略。

## [1.5.0] - 2026-02-21
- 新增排队超时自动重排功能：普通模式下排队超过指定分钟数未匹配成功时，自动退出房间并重新排队，避免长时间卡在排队状态。

## [1.4.1] - 2026-02-21
- 修复自定义阵容装备格式 bug：统一 items 为纯字符串数组，删除多余的 ItemBuild 套娃结构，解决加载阵容时 `.core is not iterable` 崩溃问题。
- 简化装备配置数据结构，移除冗余的嵌套层级。
- 优化日志等级切换胶囊的文字间距，避免文字与滑块边缘过于紧贴。


## [1.4.0] - 2026-02-15
- 优化了上重复棋子的问题。
- 现在会优先上场本等级的目标棋子。
- 新增了人机提示悬浮窗，并配置设置开关。
- 新增不可售卖的棋子类型，如魔像，训练假人不再尝试被售卖。
- 修复设置备份/恢复问题：临时备份与用户手动备份完全隔离，挂机流程不再覆盖手动备份。
- 模式选择支持 hover 浮窗，展示各模式详细信息，并支持边界检测自动调整位置。
- 支持创建自定义阵容。
- 阵容列表分栏显示：自定义阵容与默认阵容分开展示，自定义阵容置顶。
- 现在阵容列表会显示装备了。
- 模式选择hover时提供详细信息。
- 支持每局等待随机时间后再开始排队。
